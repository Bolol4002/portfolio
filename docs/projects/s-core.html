s-core — RISC-V CPU

SystemVerilog cocotb Icarus Verilog Vivado FPGA

Overview

A single-cycle RISC-V CPU implemented in SystemVerilog, built incrementally module by module. Each module is independently verified using cocotb (Python-based testbenches) and the design targets FPGA synthesis on a Xilinx Artix-7 Basys3 board.

Key Features

  • Modular, step-by-step CPU build — each module independently verified
  • RISC-V ISA-compliant instruction encoding
  • cocotb-based verification using Python (not SystemVerilog testbenches)
  • FPGA deployment target with clock divider for visual LED feedback
  • Scaffolding script (new_module.sh) for rapid module creation with boilerplate
  • OpenLane-style config.json for reproducible builds

Implemented Modules

  • ALU — 32-bit ALU supporting ADD, SUB, AND, OR with zero flag
  • Register File — 32x32-bit register file with synchronous write, async read; x0 hardwired to zero
  • Program Counter — 32-bit PC with synchronous update and reset
  • PC+4 Adder — Combinational PC+4 for sequential instruction fetch
  • Instruction Memory — 256x32-bit ROM loaded via $readmemh
  • Instruction Decoder — Extracts opcode, rd, funct3, rs1, rs2, funct7 from 32-bit instructions
  • ALU Control — Decodes R-type funct7/funct3 fields into ALU operations
  • Control Unit — Generates reg_write, alu_src, mem_read/write, mem_to_reg, branch, jump signals
  • CPU Top — Top-level integration wiring all modules into a single-cycle datapath
  • FPGA Top — FPGA wrapper with clock divider (100 MHz to ~1.5 Hz), drives 16 LEDs

Architecture

The datapath follows a classic single-cycle RISC-V design:

  • Fetch: PC indexes into instruction memory to fetch a 32-bit instruction
  • Decode: Instruction decoder extracts fields; control unit generates control signals
  • Execute: Register file reads source operands; ALU performs the operation
  • Writeback: ALU result is written back to the destination register

Currently supports R-type instructions only (ADD, SUB, AND, OR). Planned: I-type instructions, branches, jumps, and a full 5-stage pipeline.

Verification

  • Each module has its own cocotb testbench under tb/<module>/
  • Tests run via make in each testbench directory
  • Simulation performed with Icarus Verilog as the backend

FPGA Flow

  • Target: Xilinx Artix-7 (xc7a35tcpg236-1, Basys3 board)
  • Clock: 10 ns period (100 MHz), with divider to ~1.5 Hz for LED visualization
  • Constraints: Pin assignments for 16 LEDs, center button reset
  • Synthesis: Vivado batch script (create_project.tcl) for automated build
