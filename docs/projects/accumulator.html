4-bit Accumulator — ASIC Flow

Verilog OpenLane OpenROAD Yosys Magic KLayout

Overview

A 4-bit accumulator digital block implemented in Verilog, verified with a testbench, and taken through a complete ASIC physical design flow using OpenLane/OpenROAD. This project demonstrates end-to-end chip design — from RTL to GDS — with full signoff checks.

Key Features

  • Clean, minimal Verilog RTL (16 lines of design code)
  • Full testbench with VCD waveform generation
  • Complete ASIC physical design flow from RTL to GDS
  • Multiple signoff checks: DRC (Magic + KLayout), LVS, XOR, STA, IR drop
  • Reproducible flow with OpenLane config and pin order constraints

Design

  • Ports: clk (input), rst (input), en (input), data_in[3:0] (input), acc[3:0] (output)
  • Behavior: On reset, accumulator clears to zero. On rising clock edge when enabled, adds data_in to the running accumulator value. Otherwise holds.
  • Pin Config: North (clk), South (rst), East (en), West (data_in, acc)

ASIC Physical Design Flow

The project ran through 77+ OpenLane flow steps including:

  • Linting: Verilator lint, timing constructs check
  • Synthesis: Yosys synthesis, unmapped cells check, synth checks
  • Floorplanning: STA prep, floorplan, power connections, tap/endcap insertion
  • Power Distribution: PDN generation
  • Placement: IO placement, global placement, detailed placement
  • Routing: Global routing, antenna repair, detailed routing
  • Signoff: STA (pre and post PNR), IR drop report, DRC, LVS, XOR check

Manufacturing Outputs

  • GDS — Magic and KLayout layout exports
  • LEF/DEF — Library exchange format and design exchange format
  • SPICE netlist — Extracted netlist for LVS
  • SPEF/SDF — Parasitic extraction and standard delay format
  • LIB/SDC — Liberty timing and Synopsys design constraints
